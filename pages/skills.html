🛠️ AI Skills & Resources

How skills work, how to create them, and how to invoke them with slash commands

🧠 What Are Skills?

Skills are specialized instruction bundles that teach an AI agent how to perform a specific task or follow a domain-specific workflow. In this project, skills live in the project root as markdown files and provide context-aware guidance to different AI tools.

Think of a skill as a "cheat sheet" loaded into the agent's context window. When a skill is active, the agent follows its rules, conventions, and examples before generating output.

. ├── claude.md # Claude-specific guidelines ├── gemini.md # Gemini-specific guidelines ├── mimo.md # Mimo-specific guidelines └── agents.md # Multi-agent guidelines

✏️ How to Create a Skill

A skill is simply a Markdown file with structured guidance. Follow these steps:

  • Pick a scope: Decide what the skill controls (e.g., coding style, testing rules, UI patterns).
  • Name the file: Use the agent or tool name (e.g., claude.md, gemini.md).
  • Place it correctly: Drop it into the project root directory.
  • Write concise sections: Use headers (##) for clear separation. Bullet points beat paragraphs.
  • Include examples: Show the agent what "good" looks like with code snippets or decision trees.

Example: Minimal Skill Template

# Agent-Name — Project Guidelines

## 🛠️ Core Architecture
- Rule 1
- Rule 2

## 🎨 Styling & UI
- Color tokens
- Responsive breakpoints

## 📚 Content & Structure
- Directory conventions
- Naming patterns

## 🤖 AI Interaction Rules
- Surgical edits
- Verification steps

⌨️ Calling Skills with / Commands

Most AI tools let you invoke skills directly from the chat input using slash commands. The exact syntax depends on the platform:

/skill name Kilo / OpenCode — loads the named skill into the current session context.
@agent-name Some IDEs — mention the agent file to pull its instructions inline.
#file.md Generic Markdown reference — attach the skill file as context.
💡
Pro tip: If your tool supports auto-discovery, simply placing CLAUDE.md in the project root is enough. The agent reads it automatically on every request without needing a slash command.

📏 How Big Can claude.md Be?

There is no hard file-size limit, but you are constrained by the model's context window. The entire skill file is injected into the prompt on every turn, so brevity pays off.

🧰 What Should Go Inside claude.md?

Treat the file as a single source of truth for any decision the AI must make repeatedly. Prioritize high-signal, low-noise content:

  • Architecture rules — Single-file vs. multi-file, build step yes/no, CDN vs. npm.
  • Styling tokens — Hex values, spacing scale, breakpoint rules, dark-mode constraints.
  • Directory map — Where assets, data, docs, and pages live and what belongs in each.
  • Naming conventions — Question IDs (CAT01-Q005), file prefixes, branch names.
  • Interaction rules — Surgical edits, when to use edit vs write, test-before-push policy.
  • Forbidden patterns — Things the agent must never do (e.g., "Do not introduce Webpack").
  • Verification checklist — Steps to confirm a change (run tests, check live demo, lint).
🎯
Golden rule: If you find yourself correcting the same agent mistake twice, that rule belongs in claude.md.
