BMAD

Build, Model, Apply, Deliver — The Agentic Agile Framework

From Agile to Agentic Agile

BMAD is a framework for building software where AI agents don't just assist — they own entire phases of the development lifecycle. It connects business requirements to working software through orchestrated agent teams.

What is BMAD?

BMAD stands for Build, Model, Apply, Deliver — a development methodology designed for the agentic era. Traditional agile has humans at every ceremony. BMAD replaces repetitive, low-judgment work with agent loops while keeping humans in the critical decision points.

Claude Agents in Each BMAD Phase

Each phase has a primary agent role. The Coordinator orchestrates; sub-agents specialize.

🎯
Build — Product Agent

Reads business requirements, generates user stories, creates acceptance criteria, breaks epics into tasks. Uses chain-of-thought to reason about priority and dependencies.

Coordinator + Research Sub-agent
🔧
Model — Architecture Agent

Designs system architecture, defines API contracts, models database schemas, creates sequence diagrams. Validates designs against constraints in CLAUDE.md.

Coordinator + Design Sub-agent
💻
Apply — Engineering Agent

Writes code, runs tests, fixes bugs, refactors. Uses TDD with validation loops. Calls MCP tools for file system, test runner, linter, and compiler.

Coordinator + Coder + Tester Sub-agents (parallel)
🚀
Deliver — DevOps Agent

Runs CI/CD pipelines, deploys to staging, monitors metrics, creates release notes, rolls back on failure. Uses circuit breaker patterns for reliability.

Coordinator + Deploy + Monitor Sub-agents

Traditional Agile vs Agentic Agile

Ceremony Traditional Agile Agentic Agile (BMAD)
Backlog Grooming PM writes stories, team estimates Agent generates stories from requirements, human approves
Sprint Planning Team picks items, discusses scope Agent decomposes tasks, suggests priority, human decides
Daily Standup Each person reports progress Agent summarizes all sub-agent status in one report
Implementation Developer writes code Agent writes code, runs tests, human reviews PR
Code Review Peer reviews PR Agent reviews first, human approves edge cases
Retrospective Team discusses what went well Agent analyzes metrics, suggests improvements, human decides
Deployment DevOps runs pipeline Agent triggers deploy, monitors, auto-rollback on failure

📜
CLAUDE.md (CAT03)

The constitution. Defines coding standards, architectural constraints, and rules agents must never violate. Keeps the entire BMAD pipeline aligned.

🔄
Agentic Loop (CAT01)

Every BMAD phase runs as a Perceive-Think-Act loop. The agent observes the current state, reasons about next steps, acts, then verifies.

🧑
Coordinator-Subagent (CAT01)

The Coordinator assigns Build/Model/Apply/Deliver tasks to specialized sub-agents. Parallel execution where possible, sequential where dependent.

🔌
MCP Tools (CAT02)

Agents call tools: file system for code, test runner for validation, deploy script for delivery, monitoring API for observability. Standardized via MCP.

📈
Validation Loops (CAT04)

Every output is checked against criteria. Code must pass tests. Architecture must meet constraints. Deployments must pass health checks. Retry on failure.

🛠
Reliability (CAT05)

Circuit breakers prevent cascade failures. Context compression keeps long sprints within token limits. Graceful degradation when tools fail.

What Needs to Happen for Full Agentic Agile

The vision is clear but the infrastructure is still being built. Here's what's needed:

1
Reliable Long-Horizon Agents

Current agents lose context over long sessions. Need persistent memory (CAT05), better context compression, and agents that can maintain state across multi-day sprints.

2
Standardized Tool Ecosystem (MCP)

Every tool in the dev lifecycle — git, CI/CD, monitoring, issue trackers — needs MCP servers. The protocol exists (CAT02), but coverage is incomplete.

3
Trust Boundaries & Human-in-the-Loop

Agents need clear escalation patterns (CAT01). When should an agent decide alone? When should it ask? Production deployments, architectural changes, and budget decisions need human gates.

4
Evaluation Benchmarks for Agents

SWE-bench and DeepSWE measure coding ability. We need benchmarks for the full BMAD cycle — can an agent plan a sprint? Design an architecture? Run a retrospective?

5
Multi-Agent Coordination Protocols

Current Coordinator-Subagent patterns are simple. Need protocols for agent negotiation, conflict resolution, and shared state management across teams of agents.

6
Observability & Debugging

When an agent team fails, you need to understand why. Distributed tracing across agent loops, decision logging, and replay capabilities for debugging.

7
Organizational Change Management

Teams need to learn how to work with agents. New roles emerge: Agent Orchestrator, Prompt Engineer, Agent QA. The agile manifesto gets an agentic addendum.

The Agentic Agile World

When BMAD is fully realized, the development lifecycle looks fundamentally different:

🚀
Sprint 0 is Instant

An agent reads the business brief, generates the backlog, designs the architecture, and scaffolds the project — in minutes, not weeks.

🔄
Continuous Agent Loops

Agents work 24/7. While humans sleep, the Apply agent implements features, runs tests, and opens PRs for morning review.

🧑
Humans Focus on Judgment

Humans do what agents can't: make ethical decisions, negotiate with stakeholders, handle ambiguity, and approve production changes.

📈
Data-Driven Retrospectives

Agents analyze commit patterns, test coverage, bug rates, and deployment metrics to suggest concrete improvements — not just "communicate better."

🔌
Tool Ecosystem Explosion

MCP servers for every SaaS tool. Agents can query Jira, update Confluence, trigger PagerDuty, and post to Slack — all through standardized protocols.

🛠
Self-Healing Systems

Production issues trigger agent loops: detect anomaly, diagnose root cause, implement fix, deploy, verify. Humans are notified, not paged.
