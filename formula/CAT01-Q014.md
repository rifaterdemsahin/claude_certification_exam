## The Question
A user reports that a chat bot "forgets" what was said three messages ago. What is the likely technical cause?

### Options
- A) The model’s temperature is too low.
- B) The API is not being sent the full conversation history with each new request.
- C) The prompt needs more XML tags.
- D) The model has reached its maximum training data limit.

## Answer
B) The API is not being sent the full conversation history with each new request.

## Explanation
LLMs are stateless; developers must include the relevant conversation history in each API call to maintain context.

## Deep Dive
Building reliable applications with Claude requires a blend of traditional software engineering principles and AI-specific techniques. Security is paramount; API keys must always be stored on the server side, and any code execution should happen in a sandboxed environment. Performance optimization, such as using streaming for better user experience or batching tool calls to reduce round-trips, is essential for production-grade apps.

The evaluation of AI systems is an ongoing process. Instead of relying on manual vibes-based testing, developers should implement evaluation workflows using diverse datasets and model-based graders. This ensures that changes to prompts or model versions don't introduce regressions and that the system handles edge cases gracefully as it scales.

## Study Tips
- Focus on security best practices, especially regarding API key storage.
- Use evaluation datasets and model graders to move beyond 'vibes' in testing.
