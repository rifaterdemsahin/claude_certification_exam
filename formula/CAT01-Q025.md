## The Question
Which of the following is a key security limitation of the built-in Code Execution tool?

### Options
- A) It can only execute Python 2.7 code.
- B) It runs in a sandbox with no network access.
- C) It cannot perform basic mathematical operations.
- D) It requires the user to manually approve every line of code.

## Answer
B) It runs in a sandbox with no network access.

## Explanation
The built-in code execution environment is sandboxed and isolated from the network to prevent malicious use.

## Deep Dive
Claude's tool use (or function calling) is a transformative feature that moves the model from a passive observer to an active participant in digital workflows. By defining a tool with a clear name, description, and JSON schema for parameters, developers enable Claude to request actions like searching a database, calling an API, or performing complex calculations. The Model Context Protocol (MCP) further standardizes this interaction, allowing for a 'plug-and-play' ecosystem where servers can expose tools to any compatible client.

In an agentic loop, tool use represents the 'Action' phase. When Claude provides a tool_use block, it is effectively pausing its internal generation to wait for the result of that action. This grounding in external reality is what allows AI agents to overcome the limitations of their training data and interact with real-time systems safely.

## Study Tips
- Understand the 'stop_reason: tool_use' flow and how the client must respond with a 'tool_result'.
- Familiarize yourself with the distinction between MCP Hosts, Clients, and Servers.
