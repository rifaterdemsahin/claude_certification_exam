## The Question
You want Claude to analyze multiple distinct datasets (e.g., sales records and customer feedback). How should you structure this in the prompt?

### Options
- A) Paste all data into a single long paragraph.
- B) Use XML tags like <sales> and <feedback> to separate the data.
- C) Put the sales data in the system prompt and feedback in the user prompt.
- D) Provide the data as a series of comma-separated values (CSV) without headers.

## Answer
B) Use XML tags like <sales> and <feedback> to separate the data.

## Explanation
XML tags provide clear boundaries for different parts of the prompt, making it easier for Claude to distinguish and reference specific data sets.

## Deep Dive
Effective prompt engineering for Claude leverages its unique architecture, which is highly responsive to structural cues like XML tags. By wrapping different sections of a prompt—such as instructions, context, and examples—in XML tags, you reduce ambiguity and help the model identify its objectives more clearly. Techniques like few-shot prompting provide 'gold standard' examples that ground the model's output in a specific format or tone, significantly improving consistency across varied inputs.

For complex tasks, 'Chain of Thought' or 'Extended Thinking' allows the model to reason through problems before arriving at a final answer. This is particularly useful for logic-heavy tasks or when generating structured output like JSON. Prefilling the assistant's response is another powerful technique to ensure valid JSON and eliminate conversational filler, making the API response easier to parse programmatically.

## Study Tips
- Always use XML tags to separate data and instructions in long prompts.
- Practice prefilling the assistant message with '{' to force clean JSON outputs.
