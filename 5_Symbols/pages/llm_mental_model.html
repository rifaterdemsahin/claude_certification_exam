🧠 LLM Mental Model

Conceptual flow and interactions for Main Agent, Hooks, Commands, Skills, and Sub-Agents

📐 Diagram

📝 Executive Summary

The diagram establishes a mental model for understanding how a Main Agent (Chat Thread) interacts with specialized modular extensions like Hooks, Commands, Skills, and Sub-Agents (specifically in the context of tools like Claude Code). The central rationale is to offload specific tasks from the Main Agent to specialized modules to avoid prompt bloat, handle token limits efficiently, and automate workflows.

🧩 Component-by-Component Rationale

1. Main Agent (Chat Thread)

Rationale: Acts as the central nervous system or orchestrator. It manages the primary conversational context but relies on external modules to execute heavy lifting, specialized logic, or automated tasks to remain efficient.

2. Hooks (Event-Triggered Code)

Interaction: Bi-directional "Trigger & Execute" relationship with the Main Agent.

Rationale: Instead of requiring the user or Main Agent to manually initiate every action, Hooks listen for lifecycle events (e.g., after prompt, before tool). They automate repetitive tasks seamlessly behind the scenes.

3. Commands (Prompt Shortcuts)

Interaction: Main Agent routes to Commands; Commands can also trigger Hooks.

Rationale: These act as action-oriented macros. By encapsulating complex instructions into short shortcuts, they avoid long prompts, preserving the Main Agent's valuable context window.

4. Skills (Knowledge / Capability)

Interaction: Main Agent activates them via "Run Bundled Scripts".

Rationale: Skills represent static, on-demand capabilities or specialized knowledge bases. Crucially, a skill "doesn't do anything itself" — it must be explicitly invoked by an active agent or command to inject new capabilities into the runtime.

5. Sub-Agents (Isolated Task Sessions)

Interaction: The Main Agent delegates a task down; the Sub-Agent executes it and uses the "Report Back" loop to return the results.

Rationale: Complex tasks are isolated into separate sessions because Sub-Agents "own context & token limits." This protects the Main Agent from hitting token caps and prevents context contamination during deep-dive tasks.

🔗 System Relationships & Constraints

The Relationships summary box at the bottom highlights the structural hierarchy and strict operational rules of the system:

Agents Skills & Commands — Gives the core reasoning engine access to tools and shortcuts.
Skills Only bundled scripts Agents or Tools Strict isolation: Skills are passive resources, not active decision-makers.

🎯 Final Conclusion

Separation of Concerns

The overarching rationale of this mental model is separation of concerns. By dividing capabilities into active orchestrators (Agents), macro shortcuts (Commands), passive capabilities (Skills), automated event listeners (Hooks), and isolated sandboxes (Sub-Agents), the system achieves maximum power with optimal token and context management.
